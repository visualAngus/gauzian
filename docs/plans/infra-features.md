# Gauzian — Infrastructure & Features Inventory

## a) Executive Summary

Gauzian is a French end-to-end encrypted (E2EE) cloud storage platform built with a modern, security-focused architecture. The system implements a zero-knowledge model where the server never sees plaintext data—all encryption/decryption operations occur client-side using RSA-4096 asymmetric encryption and AES-256-GCM symmetric encryption.

The backend is written in **Rust** using the **Axum** web framework, providing memory safety guarantees and native performance. It exposes a REST API handling authentication, file/folder management with chunked uploads, and an encrypted calendar (agenda) feature. The frontend is a **Nuxt 3** (Vue 3) application with a custom E2EE engine using the Web Crypto API.

Infrastructure runs on **Kubernetes** (self-hosted on OVH VPS) with **PostgreSQL** for metadata, **MinIO** for encrypted object storage, and **Redis** for session management and rate limiting. Observability is provided by **Prometheus** metrics and **Grafana** dashboards. The system is currently deployed at `https://gauzian.pupin.fr` with automatic CI/CD via GitHub Actions.

Key architectural strengths include: true E2EE with per-file keys, hybrid RSA+AES encryption for secure sharing, PBKDF2 key derivation with high iteration counts, and Kubernetes-ready deployment manifests. Areas requiring attention include: password hashing migration (SHA256→Argon2id for legacy users), missing scheduled background jobs, and absence of a managed KMS/HSM for key protection.
